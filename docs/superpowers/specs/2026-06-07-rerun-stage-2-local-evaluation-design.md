# Rerun.io 阶段二本地技术评测设计

## 1. 背景

Physical AI 数据层项目已经完成阶段 1 的 Rerun.io 公开资料调研。当前共识是：Rerun.io 短期适合作为实验底座和参考架构，中期应在外围建立 CavLAB 自有数据规范和封装层，长期保留替换权。

阶段 2 进入本地技术评测。本阶段不接入机器人真机，先用模拟/仿真数据和开源机器人数据验证 Rerun 的多模态数据层能力。

## 2. 核心判断

用户提出的判断是正确的：阶段 2 不应急于接真机，而应先使用模拟或开源数据验证 Rerun 的能力边界。

原因：

- 当前要验证的是数据层能力，不是设备协议能力。
- 真机接入会引入机器人控制器、现场网络、设备权限、采样频率和安全风险，容易遮蔽 Rerun 本身的评测结论。
- 模拟数据可以主动覆盖我们关心的场景：图像、点云、TCP 位姿、焊缝轨迹、工艺参数、事件、质量结果、模型输出和训练导出。
- 开源机器人数据可作为现实数据形态对照，验证 Rerun 面对真实记录数据时的导入、对齐、查询和导出表现。

## 3. 阶段目标

阶段 2 的目标是回答一个核心问题：

> Rerun.io 是否能够支撑 Physical AI 数据层第一批样板场景所需的记录、回放、查询、整理和导出能力？

具体目标：

- 验证 Rerun 对机器人焊接工站核心数据类型的表达能力。
- 验证多 timeline、坐标系、事件、任务上下文和工艺参数能否被清晰组织。
- 验证 Viewer 是否能完成同步回放、空间查看、时间轴复盘和问题定位。
- 验证 `.rrd`、Catalog、DataFrame 查询和导出能力是否能支撑数据集整理。
- 验证 external importer 或等价导入流程是否适合未来现场数据接入。
- 形成阶段 2 技术评测报告，更新二次开发路线判断矩阵。

## 4. 非目标

本阶段不做以下事情：

- 不接入机器人真机。
- 不接 PLC、焊机、工业相机或客户现场系统。
- 不承诺完成生产级 SDK。
- 不 fork 或深度修改 Rerun 源码。
- 不做完整权限、审计、脱敏和客户现场部署方案。
- 不把 Rerun Hub 商业能力当作开源能力进行产品承诺。

## 5. 推荐路线

采用“自造焊接工站模拟数据 + 开源机器人数据对照验证”的混合路线。

### 5.1 主线：自造焊接工站模拟数据

用 Python 生成一个最小但业务语义清晰的焊接工站数据集，并通过 Rerun SDK 写入 `.rrd`。

模拟对象：

- 机器人本体；
- TCP 位姿；
- 焊缝轨迹；
- 工件坐标系；
- 相机图像；
- 简化点云；
- 工艺参数；
- 模型输出；
- 异常事件；
- 质量结果。

这条主线用于验证我们真正关心的数据模型和业务闭环。

### 5.2 对照：开源机器人数据

选择一个开源机器人数据集或仿真 demo 作为对照。优先级如下：

1. 焊接或工业机器人相关开源资料；
2. UR 系列机械臂、ROS / MoveIt / Gazebo 相关数据或 demo；
3. 具有相机、轨迹、动作和状态的通用机器人操作数据集；
4. LeRobot 生态中结构清晰、易下载的小数据集。

对照数据不要求是焊接，但应尽量覆盖机器人状态、轨迹、图像、动作或传感器数据。

### 5.3 判断原则

- 模拟数据用于验证“我们想表达什么”。
- 开源数据用于验证“Rerun 面对真实数据形态时能否处理”。
- 如果两者结论冲突，以真实数据形态暴露的问题作为下一轮重点。

## 6. 模拟数据设计

### 6.1 任务设定

模拟一个机器人焊接工站完成一条焊缝作业：

- 工件固定在工站坐标系下；
- 机器人 TCP 沿焊缝轨迹运动；
- 相机周期性采集图像；
- 点云表示工件和焊缝附近几何；
- 工艺参数随时间变化；
- 某一段出现异常事件；
- 模型输出预测质量风险；
- 作业结束后生成质量结果。

### 6.2 数据类型

| 数据 | 示例内容 | Rerun 评测点 |
| --- | --- | --- |
| TCP 位姿 | 位置、姿态、速度 | Transform、轨迹、timeline |
| 焊缝轨迹 | 规划路径、实际路径 | 3D line、对比回放 |
| 图像 | 模拟相机图像、检测覆盖层 | Image、annotation、时间同步 |
| 点云 | 工件表面、焊缝附近点 | Points3D、空间回放 |
| 工艺参数 | 电流、电压、送丝速度、焊接速度 | 时间序列、DataFrame 查询 |
| 事件 | 起弧、收弧、异常、人工确认 | timeline event、日志 |
| 模型输出 | 缺陷概率、风险区域 | scalar、annotation、overlay |
| 质量结果 | OK/NG、缺陷类型、位置 | 静态/结果数据、导出 |
| 任务上下文 | 工件 ID、焊缝 ID、程序号、批次 | static data、自有 schema |

### 6.3 时间线

至少验证以下 timeline：

- `log_time`：Rerun 默认记录时间；
- `sim_time`：模拟作业时间；
- `robot_tick`：机器人控制周期；
- `camera_frame`：相机帧序号；
- `weld_phase`：作业阶段或事件序列。

### 6.4 坐标系

至少建立以下坐标关系：

- `/world`
- `/station`
- `/workpiece`
- `/robot/base`
- `/robot/tcp`
- `/camera/front`
- `/weld/seam_001`

这套命名同时用于验证 Rerun entity path，也为后续 CavLAB 自有命名规范打底。

## 7. 实验能力清单

### 7.1 必做实验

- 生成最小焊接工站 `.rrd`。
- 用 Rerun Viewer 打开并检查多模态同步回放。
- 验证 TCP 位姿、轨迹、图像、点云、工艺参数和事件能否按时间线组织。
- 保存或记录 Blueprint 使用方式。
- 启动本地 OSS catalog，注册至少两个 simulated recordings。
- 用 Catalog SDK 或 DataFrame 查询导出关键字段。
- 形成阶段 2 技术评测报告。

### 7.2 应做实验

- 模拟异常事件并验证回放定位。
- 模拟模型输出和人工质量结果的对比。
- 尝试 external importer，将 CSV/JSON/图像目录转换为 Rerun recording。
- 使用开源机器人数据做一次对照导入或转换。

### 7.3 可选实验

- 尝试 LeRobot 导出。
- 尝试 PyTorch DataLoader。
- 做轻量性能冒烟测试：记录文件大小、加载时间、内存占用和回放流畅度。

## 8. 交付物

阶段 2 完成时应至少交付：

- 更新后的二次开发路线判断矩阵；
- 更新后的 README 和 details。

## 9. 成功标准

阶段 2 的成功标准不是“Rerun 完美可用”，而是形成足够可靠的技术判断。

最低成功标准：

- 能生成并打开一个模拟焊接工站 `.rrd`。
- Rerun Viewer 中能看到并同步回放图像、点云、TCP 位姿、轨迹、工艺参数和事件。
- 至少能通过本地 catalog 或 DataFrame 查询导出一组训练/评测候选字段。
- 能明确列出 Rerun 适合直接复用、适合封装、需要二开、必须自研和暂缓判断的能力。

理想成功标准：

- 能处理至少一个开源机器人数据源。
- 能形成 external importer 原型。
- 能导出一个最小训练样本格式。
- 能得到初步性能记录。

## 10. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 找不到合适焊接开源数据 | 影响真实感 | 用自造焊接语义数据作为主线，开源机器人数据只做对照 |
| Rerun 安装或 viewer 依赖卡住 | 影响实验效率 | 优先 Python SDK 和 `.rrd` 离线输出，必要时记录环境问题 |
| 数据模型过度模拟 | 结论不贴近现场 | 保持焊接工站语义，后续阶段 3 用真实场景校正 |
| Catalog / Dataloader API 不稳定 | 影响训练链路判断 | 标注 experimental，不作为近期产品承诺 |
| 实验范围膨胀 | 拖慢阶段推进 | 先完成最小闭环，再扩展开源数据和 importer |

## 11. 后续实施方式

确认本设计后，进入 implementation plan 阶段。计划应拆成小任务：

1. 建立实验目录和 Python 环境；
2. 写最小 Rerun SDK 冒烟脚本；
3. 生成模拟焊接工站数据；
4. 写入 `.rrd` 并验证 Viewer；
5. 增加多 timeline、Transform、事件和工艺参数；
6. 验证 Catalog / DataFrame 查询；
7. 选择并导入一个开源机器人数据源；
8. 形成阶段 2 技术评测报告；
9. 更新路线判断矩阵、README 和 details。

## 12. 待用户确认

- 本阶段主线是否确认为“模拟焊接工站数据 + 开源机器人数据对照验证”。
- 第一轮实现是否只做 Python，不引入 ROS/Gazebo/MoveIt 环境。
- 开源数据对照是否允许先选择“非焊接但数据结构接近”的机器人数据集。
